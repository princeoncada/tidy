## Locked Workflow Rules

- Radix dropdown menus should be dismissed with Escape in E2E helpers when the menu is open; trigger re-clicks can be intercepted by the document-level dismiss overlay.
- Inline rename helpers should re-resolve the active inline input after edit mode opens instead of continuing through a card-scoped locator captured before the optimistic-to-canonical row swap.
- Tests that reload after mutations must await the mutation POST before reloading; `page.reload()` can abort in-flight mutation requests and leave server state unchanged.

## Known Issues

- Inline list/item rename uses component-local edit state. The row remount on optimistic-to-canonical swap can drop an in-progress manual rename. Authenticated E2E works around this by re-resolving the inline input and awaiting mutation persistence; a product fix using stable row identity or lifted edit state remains a candidate follow-up.
- `listItem.reorderListItems` does not validate target `listId` ownership before its raw SQL update. A missing or foreign target list can surface as Postgres FK error 23503; this is tracked as 1.5.2 - Reorder Target List Ownership Fix.
